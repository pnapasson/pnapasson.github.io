หลักการของ GA4 Closed Funnel

Closed Funnel (ฟันเนลแบบปิด) คือการวิเคราะห์ที่ผู้ใช้ จะต้องเข้าสู่ Funnel ผ่าน Step 1 เสมอ และต้องทำกิจกรรม (Event) ใน Step ถัดๆ ไป ตามลำดับเวลาอย่างเคร่งครัด ภายใน Session เดียวกัน หากผู้ใช้งานข้าม Step ใดไป หรือเข้ามาเริ่มพฤติกรรมที่กลางทาง จะไม่ถูกนับรวมในผลลัพธ์ของ Funnel นี้ (เชื่อมโยงข้อมูลด้วย user_pseudo_id และ ga_session_id)

การตั้งค่า Ecommerce Logic

  • Ecommerce Logic (No): ประหยัด Cost บน BigQuery (ไม่มีการ UNNEST Items) เหมาะสำหรับ Event ทั่วไป เช่น screen_view
  • Ecommerce Logic (Yes): ดึงข้อมูลรายการสินค้า (Items) ออกมาให้ และ หาก Step ติดกันเป็น Yes ทั้งคู่ ระบบจะบังคับให้ item_id ต้องตรงกัน เพื่อให้มั่นใจว่า User ทำรายการกับสินค้าตัวเดิมต่อ
  • หมายเหตุ: Item Parameters ที่เลือกด้านล่างสุด จะถูกนำมาแสดงผลใน Query ทุก Step ที่ตั้งค่า Ecommerce Logic = Yes เพื่อให้เปรียบเทียบข้อมูลสินค้าในแต่ละขั้นตอนได้ (ส่วน Revenue จะดึงเฉพาะ Step ที่เป็น Event `purchase`)

คู่มือการใส่ค่า Dimension / Metric

  • Standard Schema: พิมพ์ชื่อปกติ เช่น event_name, device.category
  • Event Parameters: นำหน้าด้วย ep. (เช่น ep.firebase_screen) | ใช้ ep_int. หรือ ep_num. สำหรับตัวเลข
  • User Properties: นำหน้าด้วย up., up_int., หรือ up_num.
  • E-commerce Items: นำหน้าด้วย item. (เช่น item.item_category)
